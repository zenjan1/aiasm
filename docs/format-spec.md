# AI-ASM格式规范

## 概述

AI-ASM是一种严格格式化的x86_64汇编语言编写规范，专为AI生成的代码设计。基于GNU assembler (GAS) Intel语法模式，确保一致性和机器可读性。

## 基本要求

每个 `.asm` 文件必须以以下指令开头：

```asm
    .intel_syntax noprefix
```

## 函数头部注释

每个函数必须以标准注释块开头：

```asm
# -----------------------------------------------------------------------------
# 函数：函数名
# 功能：一句话描述函数功能
# 输入：
#   rdi - 参数1名称 : 参数1类型
#   rsi - 参数2名称 : 参数2类型
# 输出：
#   rax - 返回值名称 : 返回值类型
# 破坏的寄存器：rax, rcx, rdx
# 栈使用：16字节
# -----------------------------------------------------------------------------
```

## 代码格式规则

| 规则 | 说明 |
|------|------|
| 注释 | 使用 `#` 开头（GAS格式） |
| 缩进 | 使用4空格，不使用制表符 |
| 指令左对齐 | 所有指令从行首开始 |
| 函数声明 | 使用 `.globl` 而非 `global` |
| 数据定义 | 使用 `.byte` 而非 `db` |
| 长度计算 | 使用 `label_len = . - label` 而非 `equ` |
| 函数分隔 | 函数之间用空行分隔 |
| 行宽 | 建议不超过80字符 |

## RIP相对寻址

在64位模式下访问数据必须使用RIP相对寻址：

```asm
    lea     rsi, [rip + msg]
    mov     edx, msg_len
```

## 内存操作数大小

必须明确指定操作数大小：

```asm
    mov     byte ptr [rsp + 1], 10    # 正确
    mov     qword ptr [rbp - 8], 123  # 正确
```

## Linux x86_64系统调用

| 调用号(rax) | 名称 | rdi | rsi | rdx |
|-------------|------|-----|-----|-----|
| 1 | write | fd | buf | len |
| 60 | exit | status | - | - |
| 0 | read | fd | buf | len |

## syscall注意事项

`syscall` 指令会覆盖 `rcx` 和 `r11` 寄存器。在循环中使用syscall时，需要保存 `rcx`：

```asm
    push    rcx
    # ... syscall ...
    pop     rcx
```
